# Branding Workflow — NAI MVPs

## Goal
Every MVP is branded to a specific business/org/lead (no generic branding).

## Workflow
1) Select target (business/org/lead).
2) Create a 3‑line brand brief:
   - Who they are
   - Pain + why it matters
   - Promise/outcome
3) Update MVP assets:
   - Hero headline + subhead
   - CTA wording
   - Use‑case bullets
   - Logo/brand accents (if applicable)
4) Log in `BRANDING_STATUS.md` + `MVP_BRANDING_PLAN.md`.
5) If multiple targets: duplicate MVP and brand each variant.

## Done Criteria
- Target name present on page
- CTA speaks to target’s outcomes
- Copy avoids generic language
