# Website Polish Workflow — New Ashtabula Initiative

## The Process (Per Website)

### Phase 1: Research & Recon
- [ ] Identify the actual business/entity
- [ ] Capture branding (colors, logo, voice)
- [ ] Research competitors and pricing
- [ ] Document current pain points
- [ ] **Output:** `research.md` in site folder

### Phase 2: Resource Procurement
- [ ] Gather required documents (zoning PDFs, permits, etc.)
- [ ] Source relevant data/APIs
- [ ] Collect reference materials
- [ ] **Output:** `resources/` folder with downloaded assets

### Phase 3: Specification
- [ ] Define exact functionality (no generic LLM fluff)
- [ ] Wireframe/key screens
- [ ] Content requirements
- [ ] Integration specs
- [ ] **Output:** `SPEC.md` — the build blueprint

### Phase 4: Build & Polish
- [ ] Custom design matching brand
- [ ] Real functionality (not placeholder)
- [ ] Live data where possible
- [ ] Mobile-first, fast, accessible
- [ ] **Output:** Deployed site that's impossible to refuse

### Phase 5: Deploy & Pitch
- [ ] Deploy to live URL
- [ ] Prepare pitch deck
- [ ] Contact business with "it's already built" offer
- [ ] **Output:** `PITCH.md` + live demo
