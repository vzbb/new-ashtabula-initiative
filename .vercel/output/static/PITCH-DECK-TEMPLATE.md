# Pitch Deck Template — New Ashtabula Initiative

**Purpose:** Standardized presentation for pitching civic tools to stakeholders  
**Format:** Reusable Google Slides / PowerPoint / Keynote outline  
**Audience:** City officials, business owners, community organizations  

---

## Slide Structure (12 Slides)

### 1. Title Slide
**Content:**
- Tool Name (e.g., "permit-whisperer")
- Tagline (e.g., "Making permits simple for Ashtabula residents")
- Noirsys AI logo
- Date

**Visual:** Clean, dark theme with node network background (subtle)

---

### 2. The Problem
**Headline:** "Residents struggle with [specific pain point]"

**Content:**
- 2-3 specific pain points (from research)
- Quantify if possible ("30-day review feels opaque", "Two locations to visit")
- Resident quote (if available)

**Visual:** Split screen — frustrated resident vs. confusing process

---

### 3. Current State
**Headline:** "How it works today"

**Content:**
- Step-by-step current workflow
- Time required
- Common failure points
- Cost to resident (time + money)

**Visual:** Flowchart showing friction points

---

### 4. The Solution
**Headline:** "[Tool Name] — [one-sentence value prop]"

**Content:**
- What it does (3 bullet points max)
- Who it's for
- Key benefit (time saved, clarity, accessibility)

**Visual:** Screenshot/mockup of tool interface

---

### 5. Demo / Screenshots
**Headline:** "See it in action"

**Content:**
- 3-4 key screens
- Walk through main user flow
- Highlight ease of use

**Visual:** Actual screenshots or high-fidelity mockups

---

### 6. Impact Metrics
**Headline:** "Measurable benefits"

**Content:**
| Metric | Before | After | Improvement |
|--------|--------|-------|-------------|
| Time to understand permits | 2+ hours | 5 minutes | 95% faster |
| Forms to download | 15+ scattered | 1 unified | Simplified |
| Office visits | 2-3 | 0-1 | Reduced |

**Visual:** Before/after comparison chart

---

### 7. Technical Approach
**Headline:** "Built with modern, reliable tech"

**Content:**
- Stack: Next.js, deployed to Vercel
- Mobile-responsive design
- No server maintenance required
- Fast, accessible, secure

**Visual:** Simple tech stack icons

---

### 8. Investment Required
**Headline:** "What we need from you"

**Content:**
| From Stakeholder | From Noirsys |
|------------------|--------------|
| Logo/brand usage | Free tool development |
| Data verification | Ongoing maintenance |
| Promotion to residents | Analytics + improvements |
| Feedback loop | Training materials |

**Visual:** Two-column comparison

---

### 9. Success Metrics
**Headline:** "How we'll measure success"

**Content:**
- Usage metrics (monthly active users)
- Resident satisfaction (survey)
- Staff time saved (if applicable)
- Adoption rate targets

**Visual:** Dashboard mockup showing metrics

---

### 10. Timeline
**Headline:** "From prototype to production"

**Content:**
| Phase | Duration | Deliverable |
|-------|----------|-------------|
| Pilot | 2 weeks | Working demo |
| Feedback | 2 weeks | Iteration |
| Launch | 1 week | Public deployment |
| Scale | Ongoing | Additional features |

**Visual:** Gantt-style timeline

---

### 11. Risk Mitigation
**Headline:** "Addressing concerns upfront"

**Content:**
| Concern | Mitigation |
|---------|------------|
| "What if it's wrong?" | Clear disclaimers, official sources only |
| "Who maintains it?" | Noirsys committed to 12-month support |
| "What about mobile?" | Fully responsive design |
| "Is data secure?" | No PII collection, analytics only |

**Visual:** Table format

---

### 12. Call to Action
**Headline:** "Let's make this real"

**Content:**
- Specific ask (e.g., "Approve logo usage", "Partner for pilot", "Introduce to [person]")
- Next step (e.g., "2-week pilot starting [date]")
- Contact: Michael A. Vega, contact@noirsys.com

**Visual:** Clear CTA button, contact QR code

## Design Guidelines

### Colors (Noirsys Brand)
- Primary: `#0a0a0f` (deep black)
- Accent: `#3b82f6` (electric blue)
- Text: `#e2e8f0` (light gray)
- Highlight: `#10b981` (success green)

### Fonts
- Headlines: Inter Bold
- Body: Inter Regular
- Monospace: JetBrains Mono (for code/metrics)

### Visual Style
- Dark theme with subtle node network background
- Glass-morphism cards for content
- Generous whitespace
- Animated transitions (subtle)

---

## Delivery Tips

1. **Keep it under 10 minutes** — Respect stakeholder time
2. **Lead with problem** — Make it relatable
3. **Show, don't tell** — Demo > description
4. **Address objections early** — Build trust
5. **End with clear ask** — Don't leave ambiguous
