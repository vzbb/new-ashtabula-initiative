# Permit-Whisperer — Deployment Checklist

## Pre-Launch Checklist

### Technical
- [ ] Final responsive testing (mobile/tablet/desktop)
- [ ] Performance audit (Lighthouse >90 all categories)
- [ ] Accessibility check (WCAG 2.1 AA)
- [ ] Cross-browser testing (Chrome, Safari, Firefox, Edge)
- [ ] SEO meta tags + Open Graph
- [ ] Favicon + app icons
- [ ] Analytics integration (Google Analytics or Plausible)
- [ ] Error tracking (Sentry or similar)
- [ ] Backup strategy documented

### Content
- [ ] Fee schedule finalized and accurate
- [ ] All forms linked and accessible
- [ ] Contact info verified
- [ ] "Last updated" date visible
- [ ] Disclaimer/legal text reviewed

### Outreach
- [ ] Stakeholder emails sent (see outreach-package/)
- [ ] Responses tracked
- [ ] Demo scheduled with city officials
- [ ] Feedback incorporated

### Deployment
- [ ] Domain configured
- [ ] SSL certificate active
- [ ] Firebase deploy tested
- [ ] Custom domain connected
- [ ] Uptime monitoring enabled

## Post-Launch (Week 1)
- [ ] Monitor analytics daily
- [ ] Collect initial user feedback
- [ ] Watch for error reports
- [ ] Social media announcement
- [ ] Email stakeholders success metrics

## Success Metrics
- Unique visitors (target: 100/week by month 1)
- Form downloads (track via analytics)
- Bounce rate (target: <40%)
- Time on site (target: >2 min)
- City adoption (meeting scheduled)
