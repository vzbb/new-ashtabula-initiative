# Parts Finder — Project Context

## Vision
"Kayak for auto parts" — A unified search interface for auto parts availability across all local stores in Ashtabula County, with price comparison and reservation capability.

## Current Status
- **Phase:** 1 Complete (Research)
- **Prototype:** React + Gemini API functional
- **Blocker:** Store API integration (AutoZone, Advance, etc.)
- **Next:** Phase 2 Resource Procurement + Phase 3 SPEC

## Problem Statement
Finding auto parts requires calling 3-5 stores. Wasted trips when parts not in stock. No price comparison between stores. Special order delays with no visibility.

## Target Market
- **DIY mechanics:** 8,000-12,000 in Ashtabula County
- **Repair shops:** 30-50 independent shops
- **Annual parts spend:** $15M-$25M
- **Parts stores:** 8-12 locations (AutoZone, Advance, O'Reilly, NAPA, etc.)

## Prototype
- **Stack:** React + Vite + Gemini API
- **Location:** `projects/new-ashtabula-initiative/websites/parts-finder/`
- **Status:** Functional demo, needs deployment
- **Features:** AI-generated availability responses, reserve CTA
