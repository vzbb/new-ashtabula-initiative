**Top Pain Points for Small-to-Mid Contractors:**

## 7. Technical Feasibility Assessment

### Core Technology Stack
- **Frontend:** React + TypeScript + Tailwind CSS
- **Backend:** Firebase (Auth, Firestore, Storage)
- **AI Vision:** Google Gemini Vision API
- **Hosting:** Firebase Hosting / Vercel
- **File Processing:** PDF.js, Sharp (image processing)

### Technical Capabilities Required

| Feature | Tech | Feasibility | Notes |
|---------|------|-------------|-------|
| PDF Blueprint Upload | Firebase Storage + PDF.js | ✅ High | Standard implementation |
| Image preprocessing | Sharp / Canvas | ✅ High | Resize, enhance for OCR |
| AI Measurement Extraction | Gemini Vision API | ✅ High | Proven for document analysis |
| Square footage calc | Custom algorithm | ✅ High | Math based on AI output |
| Room detection | Gemini Vision + post-processing | ⚠️ Medium | Requires prompt engineering |
| Material takeoffs | AI + formula engine | ⚠️ Medium | Needs calibration per trade |
| Collaboration/sharing | Firestore real-time | ✅ High | Firebase native |
| Mobile responsiveness | React + Tailwind | ✅ High | Standard PWA approach |

### Gemini Vision API Suitability
- **Document understanding:** Native PDF support (Gemini 3)
- **Object detection:** Bounding boxes for rooms, features
- **Structured output:** JSON mode for measurements
- **Pricing:** ~$0.075-0.30 per 1K tokens (very affordable)
- **Rate limits:** 1,000-2,000 RPM on standard tier

### Technical Blockers Identified
1. **Scale calibration** — Need user input or known reference for accurate measurements
2. **Blueprint quality variance** — Hand-drawn vs. CAD-produced variations
3. **Multi-page PDF handling** — Sheet indexing and navigation
4. **Offline capability** — Limited without service worker implementation

## 8. Critical Blockers for Phase 2

### Blocker 1: Scale/Calibration Accuracy
**Risk:** HIGH — Without accurate scale reference, measurements will be wrong
**Mitigation:** Require user to specify scale (e.g., 1/4" = 1') or identify a known measurement
**Status:** Needs validation in MVP testing

### Blocker 2: AI Output Consistency
**Risk:** MEDIUM — Gemini Vision may return inconsistent measurement formats
**Mitigation:** Structured JSON output with validation schemas; prompt engineering
**Status:** Testable with prototype

### Blocker 3: Market Price Sensitivity
**Risk:** MEDIUM — Contractors may still find $29/mo too expensive
**Mitigation:** Offer annual discount (20% off); freemium to demonstrate value
**Status:** Validate in discovery calls

### Blocker 4: Data Privacy Concerns
**Risk:** MEDIUM — Contractors may not want proprietary blueprints on cloud
**Mitigation:** Clear data policy; local processing option (future); encryption
**Status:** Address in terms of service

### Blocker 5: Integration with Existing Workflows
**Risk:** LOW-MEDIUM — Need to export to common formats (Excel, PDF)
**Mitigation:** CSV/PDF export in MVP; consider QuickBooks/Xero integrations later
**Status:** Include in MVP scope
